Method

What is a Cubic Spline?

A cubic spline is a piecewise cubic polynomial that passes through a given set of data points and is smooth at every interior point — value, slope, and curvature all match across adjacent pieces.

It avoids the wild oscillations of high-degree polynomial interpolation (Runge's phenomenon), making it the standard tool for smooth interpolation in graphics, engineering, and data science.

Definition

Given $n+1$ data points $(x_0, y_0), (x_1, y_1), \dots, (x_n, y_n)$, a cubic spline $S(x)$ consists of $n$ cubic polynomials, one per interval $[x_i, x_{i+1}]$:

$$S_i(x) = a_i + b_i(x - x_i) + c_i(x - x_i)^2 + d_i(x - x_i)^3$$

Conditions

  • Interpolation: $S_i(x_i) = y_i$ and $S_i(x_{i+1}) = y_{i+1}$
  • Slope continuity: $S'_{i-1}(x_i) = S'_i(x_i)$
  • Curvature continuity: $S''_{i-1}(x_i) = S''_i(x_i)$
  • Boundary conditions (two more equations needed):
    • Natural: $S''(x_0) = S''(x_n) = 0$
    • Clamped: $S'(x_0) = f'_0$, $S'(x_n) = f'_n$ (user-supplied slopes)

Algorithm (Burden & Faires)

  1. Compute interval widths $h_i = x_{i+1} - x_i$.
  2. Set $a_i = y_i$.
  3. Build the tridiagonal system for second-derivative coefficients $c_i$: $$h_{i-1}c_{i-1} + 2(h_{i-1}+h_i)c_i + h_i c_{i+1} = 3\!\left(\tfrac{y_{i+1}-y_i}{h_i} - \tfrac{y_i - y_{i-1}}{h_{i-1}}\right)$$
  4. Solve via the Thomas algorithm (forward sweep + back substitution).
  5. Recover the remaining coefficients: $$b_i = \tfrac{y_{i+1}-y_i}{h_i} - \tfrac{h_i(2c_i + c_{i+1})}{3}, \qquad d_i = \tfrac{c_{i+1} - c_i}{3 h_i}$$

Where it's used

Computer Graphics

Smooth curves for fonts, vector illustration, and animation paths.

Engineering & CAD

Designing aerodynamic shapes — wings, hulls, car bodies — from sparse points.

Data Science

Smooth missing-value imputation, monotone calibration curves, and signal resampling.

Conclusion

Cubic spline interpolation is a clean answer to a hard problem: how to draw a smooth curve through data without the explosive oscillations that plague high-degree polynomials. By insisting on low-degree pieces (cubic on each interval) joined under $C^2$ continuity, the method trades a single global formula for a system that is both visually faithful and numerically stable — and the tridiagonal solve makes it cheap enough to run in real time on millions of points.

The wider implication is methodological. Many problems in numerical analysis — root-finding, integration, ODE solving — share the same lesson: a globally-defined object often misbehaves, while a piecewise-local construction with smoothness conditions stays well-conditioned. Cubic splines made this idea concrete in interpolation; the same pattern reappears in finite-element methods, signal processing kernels, and modern machine-learning techniques (e.g., natural cubic splines as regression smoothers, monotone splines for calibration). Learning splines, then, isn't just learning interpolation — it's a first encounter with one of the most productive ideas in computational mathematics.
